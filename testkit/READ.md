这是一个非常关键的步骤。通常在做 OS 实验时，你需要把代码提交到**你自己**的仓库（以便助教评分或自己备份），而不是推送到蒋炎岩老师的 `jyy/os2025` 仓库（因为你没有权限，直接推会报 403 错误）。

根据你之前的截图，你的 `origin` 很可能还指向老师的仓库。请按照以下步骤，把你的本地仓库关联到你自己的远程仓库：

### 第一步：准备你自己的远程仓库

如果你还没有在学校的 GitLab（或者 GitHub）上建立自己的仓库：

1.  登录学校的 Git 网站（例如 `git.nju.edu.cn`）。
2.  创建一个新的**空仓库**（通常命名为 `os2025` 或类似的）。
3.  **复制**这个新仓库的地址（推荐使用 SSH 地址 `git@...`，如果没有配置 SSH Key 则使用 HTTPS 地址 `https://...`）。

### 第二步：修改远程仓库关联（最推荐的做法）

我们需要把老师的仓库改名为 `upstream`（上游），把你自己的仓库设置为 `origin`（起源）。这样你既能拉取老师的更新，又能提交自己的代码。

在终端中依次运行以下命令：

**1. 查看当前远程仓库状态**

```bash
git remote -v
```

*(如果显示 `origin` 是 `.../jyy/os2025`，请继续往下做)*

**2. 把老师的仓库重命名为 'upstream'**

```bash
git remote rename origin upstream
```

**3. 添加你自己的仓库为 'origin'**
*(把下面的 `<你的仓库地址>` 换成你在第一步里复制的地址)*

```bash
git remote add origin <你的仓库地址>
```

*例如： `git remote add origin git@git.nju.edu.cn:your_name/os2025.git`*

### 第三步：推送到你的仓库

现在，把你的代码（包含刚才合并的 M1）推送到你新设置的 `origin`：

```bash
git push -u origin main
```

*(注意：如果你的主分支叫 `master`，请把 `main` 换成 `master`。你可以通过 `git branch` 查看当前分支名)*

-----

### 验证是否成功

1.  运行命令后，不应该报错，而是显示写入对象的进度。
2.  刷新你自己的 GitLab/GitHub 网页，你应该能看到刚才合并进来的 M1 代码以及你自己的提交。

**以后的工作流程：**

  * **拉取老师的新实验（M2, M3...）：** `git pull upstream M2`
  * **保存自己的进度：** `git add .` -\> `git commit` -\> `git push`
